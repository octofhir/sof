---
title: "FH07 — More than one iteration construct per select"
description: A single select uses more than one of forEach, forEachOrNull or repeat.
tableOfContents: false
---
**Group:** Structure · **Severity:** error · **Run by:** `validate` and `lint`

The SQL-on-FHIR v2 `sql-expressions` invariant allows **at most one** of
`forEach`, `forEachOrNull` or `repeat` on a single `select`. Combining them is
ambiguous and is rejected.

## Example

This select uses both `forEach` and `repeat` — FH07:

```json
{
  "resource": "Patient",
  "select": [{
    "forEach": "name",
    "repeat": ["link"],
    "column": [{ "name": "id", "path": "id" }]
  }]
}
```

```text
error [FH07]: a select may use at most one of forEach, forEachOrNull or repeat
```

## Fix

Keep one iteration construct per `select`; nest a child `select` if you need a
second level of iteration.
