---
title: "FH10 — Missing resource or no columns"
description: The view is missing the required resource, or produces no columns.
tableOfContents: false
---
**Group:** Structure · **Severity:** error · **Run by:** `validate` and `lint`

A ViewDefinition must declare the required `resource` and must produce at least
one output column. An empty `resource` or a view whose `select` tree yields no
columns is rejected.

## Example

No columns anywhere — FH10:

```json
{
  "resource": "Patient",
  "select": [{ "column": [] }]
}
```

```text
error [FH10]: ViewDefinition produces no columns
```

A missing/empty `resource` reports `ViewDefinition is missing the required
resource`.

## Fix

Set the `resource` (e.g. `"resource": "Patient"`) and define at least one
column.
